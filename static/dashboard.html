Why threads instead of asyncio for the provider code?
The Azure SDK and Paramiko are synchronous and blocking â€” they hold the GIL during network I/O. Forcing them into asyncio via asyncio.to_thread would work but hides that they're blocking. Explicit daemon threads make the model clear: one thread per job, lifetime tied to the operation. The WebSocket handler bridges back to async via loop.run_in_executor() â€” only the queue read crosses the boundary.
ğŸ“„
Why a flat file (state.json) instead of SQLite or Redis?
At one active VM at a time, there is no concurrency on the state file â€” only one write (provision) and one delete (destroy) ever happen. Adding a DB introduces a service dependency, migration scripts, and connection pooling for zero benefit. The file approach also survives server restarts cleanly: on startup the server reads the file and knows exactly what's deployed.
ğŸ“¡
How does real-time log streaming work without SSE or polling?
The provider pushes log lines into an in-process queue as it runs. The WebSocket handler drains that queue on the async event loop using run_in_executor() â€” this bridges the sync thread to async without blocking the server. A heartbeat ping is sent every 15 seconds to keep the connection alive. Late-joining clients receive the full log history replayed immediately on connect.
ğŸ›¡ï¸
How is the public demo protected from abuse?
Four independent guards: (1) Token-bucket rate limiter per IP â€” 4 writes/min, 60 reads/min. (2) Global concurrency cap â€” only one VM operation runs at a time, server-wide. (3) Active VM budget â€” maximum 3 live VMs, counter decrements on destroy (not a lifetime total). (4) Mandatory auto-destroy â€” a server-side timer fires after 20 minutes regardless of user action; visitors cannot disable it.

How does the provider pattern work and why?
An abstract BaseProvider interface defines provision(), deploy(), destroy(), get_status(). AzureProvider implements it. The API layer calls a factory function and never imports the cloud SDK directly. Adding AWS or GCP means creating a new provider class and registering it â€” the API, CLI, and dashboard change zero lines.
